# Agente: ux-virahit (Uma)
# Versao: 1.0 | Data: 2026-05-07
# Design System: Cream #F4EEDC | Teal #2C5D63 | Gold #EAA115
# Mobile-first: base 375px

---

## PREMISSAS DE DESIGN

- Mobile-first absoluto: 80% do trafego e mobile
- Publico: mulheres 35-55 em momento emocional (buscando presente)
- Emocao antes de logica: elevacao emocional ANTES de mostrar preco
- Atrito zero: cada campo a mais = abandono
- Estado emocional mantido do inicio ao fim do funil

## ANTI-PATTERNS IDENTIFICADOS

1. **CTA transacional ("Comprar", "Pagar")**
   - Impacto: quebra o estado emocional, lembra que e uma transacao comercial
   - Correto: "Criar minha musica personalizada", "Continuar com este plano", "Confirmar pedido"

2. **Garantia apenas no checkout**
   - Impacto: visitante abandona a landing por medo de perder dinheiro sem saber da garantia
   - Correto: garantia visivel na landing ANTES do primeiro CTA e repetida no checkout

3. **Campo de historia obrigatorio**
   - Impacto: passo 4 bloqueia avanco se vazio — usuario que nao quer escrever abandona
   - Correto: campo opcional, CTA nunca bloqueado, placeholder com exemplos encorajadores

4. **Background branco (#FFFFFF)**
   - Impacto: perde identidade editorial da marca, parece site generico
   - Correto: cream (#F4EEDC) em todas as paginas e cards

5. **Border-radius uniforme e simetrico em tudo**
   - Impacto: perda da assinatura editorial ViraHit (assimetria intencional)
   - Correto: usar .asymmetric-border (2px 40px 4px 60px) em cards de destaque e fotos

6. **Botao "Nao obrigado" oculto ou muito pequeno no upsell**
   - Impacto: dark pattern que gera desconfianca e possiveis chargebacks
   - Correto: botao de recusa visivel, clicavel, com texto claro — nunca ocultar

7. **QR Code estatico sem countdown**
   - Impacto: usuario nao sente urgencia para pagar, PIX expira sem ele saber
   - Correto: timer countdown visivel, alerta visual quando < 5 minutos restantes

8. **Sem auto-save no quiz**
   - Impacto: qualquer interrupcao (ligacao, notificacao) mata o progresso — abandono garantido
   - Correto: auto-save localStorage a cada 5s, retomar da onde parou ao voltar

## NOTAS PARA O DEV (dev-virahit)

### Landing Page
- Carousel de ocasioes: usar Embla Carousel ou similar, scroll-snap no mobile
- Videos: usar elemento <video> nativo com poster, lazy-load, sem autoplay com som
- Audio players: estado global com Zustand ou Context para evitar 2 players tocando simultaneamente
- Hero CTA: ancora para quiz via smooth-scroll, nao redirect de pagina
- Mobile: testar hero com teclado virtual aberto (reduz viewport 40%)

### Quiz
- Estado do quiz: persistir em localStorage com chave `virahit-quiz-state`
- Passo 1 (selecao): avanco automatico — nao exigir botao "continuar"
- Voice input: usar Web Speech API (fallback: desabilitar botao se nao suportado, nao erro)
- Auto-save textarea: debounce 5000ms, apenas se houve mudanca
- Plano selecionado: salvar em sessionStorage para o checkout recuperar

### Checkout
- QR Code: gerar via route handler POST /api/woovi-charge, nunca client-side
- Polling: abort controller para cancelar ao sair da pagina
- Modo discreto: Page Visibility API + visibilitychange event
- Order bump: recalcular total e recriar QR Code ao marcar/desmarcar
- Validacao de email: regex basico client-side + validacao real no servidor

### Pos-pagamento
- Dados do pedido: receber via query params criptografados ou sessionStorage
- Upsell: POST separado para /api/woovi-charge com novo valor, novo pedido Baserow
- Analytics: disparar evento de conversao ao montar a pagina (Google Analytics / Meta Pixel)

### Acessibilidade (critica para o publico 35-55)
- Tamanho minimo de fonte: 14px em qualquer contexto
- Tamanho minimo de area de toque: 44x44px (botoes, cards de opcao)
- Contraste: teal #2C5D63 sobre cream #F4EEDC = ratio 6.2:1 (WCAG AA aprovado)
- Labels de formulario: sempre visiveis, nunca apenas placeholder
- Focus visible: nunca remover outline — usar ring gold customizado

### Performance (critica para conversao)
- LCP meta: < 2.5s no mobile 4G
- Imagens: next/image com priority no hero, lazy para o resto
- Fontes: preload Open Sans 800 e Merriweather, font-display: swap
- Bundle: code-splitting por rota (landing / quiz / checkout / obrigado)
- Tailwind: purge agressivo em producao
